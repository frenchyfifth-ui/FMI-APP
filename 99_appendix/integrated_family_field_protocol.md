# Integrated Family Field Protocol (v1.0)
## Human Continuity Under Constraint

### Purpose
A minimal, reproducible protocol to maintain human functionality, social presence, and execution capacity under sustained cognitive and systemic load.

This is an operational guide, not a belief system.

---

## Core Operating Rules

### Rule 1 — One Mode at a Time
System operates in exactly one mode:
- THINK
- BUILD
- MAINTAIN
- HUMAN

Mode switching is explicit. Cross-mode activity is treated as an error.

---

### Rule 2 — Daily Alignment (15 minutes)
Once per day, all participants answer:
- One completed action (yesterday)
- One executable priority (today)
- One constraint or request for support

No discussion. No optimization.

---

### Rule 3 — Execution Before Ideation
New ideas are not allowed unless an artifact exits the system first.
Artifacts include:
- A written page
- A committed file
- A completed task visible to others

---

### Rule 4 — Hard “Done”
Every task must define:
`DONE = observable, external, boring`

If it still feels intellectually exciting, it is not done.

---

### Rule 5 — Minimum Human Baseline (Non-Optional)
Regardless of system state:
- Sleep
- Eat
- Move
- Speak to another human

Skipping these guarantees instability.
